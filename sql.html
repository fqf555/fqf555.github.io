基本概念

数据库 (DB)、数据库管理系统 (DBMS)、数据库系统管理员 (DBA)、数据库系统 (DBS)

码（超码）：能够唯一确定一行的一组属性

主码：任一真子集都不是超码的超码（最小超码）

数据库对象：组成数据库的逻辑成分

SQL Server 的数据库对象主要包括：表、视图、索引、存储过程、触发器和约束等

完全限定名：

服务器名.数据库名.数据库架构名.对象名

server.database.scheme.object

（scheme 一般写 dbo）

部分限定名：

完全限定名的四个部分中的前三个部分均可省略，省略部分使用如下默认值：

server: 默认为本地服务器

database: 默认为当前数据库

scheme: 默认为 dbo

sa: superuser in SQL Server

  • DDL（数据定义语言）：就是 CREATE, ALTER, DROP
  • DML（数据操纵语言）：就是 INSERT, UPDATE, DELETE
  • DCL（数据控制语言）

SQL Server 使用的文件：

  • 主数据文件：每个数据库必须有且仅能有一个主文件。默认扩展名为 .mdf
  • 辅助数据文件：默认扩展名为 .ndf
  • 日志文件：每个数据库至少有一个日志文件，也可以有多个。默认扩展名为 .ldf

SQL Server 中有两类文件组：主文件组、用户定义文件组（课本P24）

SQL Server 的系统数据库：

  • master：管理数据库的数据库
  • model：为新创建的数据库提供模板的系统数据库
  • msdb：定时计划任务（例如每天几点自动备份和转储等）
  • tempdb：存放 T-SQL 执行过程中借助的临时表

SQL Server 支持的数据类型：（见课本 P42-47）

注意：T-SQL 中，字符串必须用单引号，双引号会被认为是对象名（如列名、表名）

一些简单的例子

【例】查询表 XSB 的所有数据：

SELECT * FROM XSB;

SELECT 学号, 姓名, 总学分
FROM XSB

给结果集取别名

xxxxxxxxxx
WITH table1 AS (
    SELECT * FROM XSB);

列别名

【例】查询 XSB 表中计算机系同学的学号、姓名和总学分，结果中各列的标题分别制定为 no、name 和 mark：

（以下三种写法等价）

SELECT 学号 AS no, 姓名 AS name, 总学分 AS mark
FROM XSB
WHERE 专业='计算机';
xxxxxxxxxx
SELECT 学号 no, 姓名 name, 总学分 mark
FROM XSB
WHERE 专业='计算机';
xxxxxxxxxx
SELECT no=学号, name=姓名, mark=总学分
FROM XSB
WHERE 专业='计算机';

注意：不允许在 WHERE 子句中使用刚刚定义的列别名，因为执行 WHERE 代码时可能尚未确定列值。例如，下述查询都是非法的：

xxxxxxxxxx
SELECT 性别 AS sex
FROM XSB
WHERE sex=0;
xxxxxxxxxx
SELECT 性别 sex
FROM XSB
WHERE sex=0;
xxxxxxxxxx
SELECT sex=性别
FROM XSB
WHERE sex=0;

CASE: 替换查询结果中的数据

CASE – WHEN – THEN – ELSE – END

【例】查询 XSB 表中计算机系各同学的学号、姓名和总学分，并对其总学分按等级进行相应替换：

xxxxxxxxxx
SELECT 学号, 姓名, 等级=
    CASE
        WHEN 总学分 IS NULL THEN '尚未选课'
        WHEN 总学分 < 50 THEN '不及格'
        WHEN 总学分 >= 50 AND 总学分 <= 52 THEN '合格'
        ELSE '优秀'
    END
FROM XSB
WHERE 专业='计算机';

DISTINCT: 去重

【例】对 XSB 表中只选择专业和总学分，消除结果集中的重复行：

SELECT DISTINCT 专业, 总学分
FROM XSB;

若不加 DISTINCT，默认为 ALL（不去重）。

TOP: 限制结果集返回行数

TOP 子句可以用于 SELECT, INSERT, UPDATE 和 DELETE 语句中。

【例】对 XSB 表选择姓名、专业和总学分，返回结果集的前 6 行：

SELECT TOP 6 姓名, 专业, 总学分
FROM XSB;

【例】对 XSB 表选择姓名、专业和总学分，返回结果集的前 10%：

SELECT TOP 10 PERCENT 姓名, 专业, 总学分
FROM XSB;

WITH TIES: 与 ORDER BY 连用，最后并列的结果全部返回。

【例】查询总学分前 10 名的同学的全部信息，如果最后有并列结果则全部返回。

xxxxxxxxxx
SELECT TOP 10 WITH TIES *
FROM XSB
ORDER BY 总学分 DESC;

聚合函数

常用聚合函数：

AVG, COUNT, MAX, MIN, SUM

用聚合函数时不一定用 GROUP BY。

聚合函数的用法举例：

【例】求选修 101 课程的学生的平均成绩：

SELECT AVG(成绩) AS '课程101平均成绩'
FROM CJB
WHERE 课程号='101';

WHERE 子句

LIKE / NOT LIKE: 字符串模式匹配

用作字符串比较时，”=” 不能使用通配符，”LIKE” / “NOT LIKE” 可以使用通配符。

通配符列表：

  • %: 相当于 *
  • _: 相当于 ? （注：汉字也算1个”_”）
  • [a-f]: 指定范围
  • [^a-f]: 指定不属于范围

【例】查询 XSB 表中学号倒数第3个数字为1，且倒数第1个数在1至5之间的学生的学号、姓名及专业。

SELECT 学号,姓名,专业
FROM XSB
WHERE 学号 LIKE '%1_[12345]';

如果需要查找的模式串中含通配符，则必须使用转义字符。

【例】查询 XSB 表中名字含下划线的学生学号和姓名。

-- 法一：把要查找的通配符写在中括号里
​
SELECT 学号,姓名
FROM XSB
WHERE 学号 LIKE '%[_]%';
​
-- 法二：使用 ESCAPE
​
SELECT 学号,姓名
FROM XSB
WHERE 学号 LIKE '%#_%' ESCAPE '#';

ISNULL 函数

ISNULL(总学分, 0): 若总学分的值不为 NULL，就返回原值；若为 NULL，则用 0 代替之。

【例】查询 XSB 表中通信工程专业总学分不足 42 的同学的情况：

xxxxxxxxxx
SELECT *
FROM XSB
WHERE 专业='通信工程' AND ISNULL(总学分, 0) < 42;

BETWEEN … AND …

BETWEEN value1 AND value2 是闭区间 [value1, value2]。

【例】查找 XSB 中不在 1989 年出生的学生情况。

xxxxxxxxxx
SELECT *
FROM XSB
WHERE 出生时间 NOT BETWEEN '1989-1-1' AND '1989-12-31';

嵌套子查询

【例】查找未选修离散数学的学生情况。

SELECT *
FROM XSB
WHERE 学号 NOT IN (
    SELECT 学号
    FROM CJB
    WHERE 课程号 IN (
        SELECT 课程号
        FROM KCB
        WHERE 课程名='离散数学'
    )
);

ALL, SOME/ANY

SOME 或 ANY 表示表达式只要与子查询结果集中的某个值满足比较的关系，就返回 TRUE，否则返回 FALSE。

【例】查找比所有计算机系的学生年龄都大的学生。

xxxxxxxxxx
SELECT *
FROM XSB
WHERE 出生时间 < ALL (
    SELECT 出生时间
    FROM XSB
    WHERE 专业='计算机'
);

等价写法：使用聚合函数 MIN

xxxxxxxxxx
SELECT *
FROM XSB
WHERE 出生时间 < (
    SELECT MIN(出生时间)
    FROM XSB
    WHERE 专业='计算机'
);

【例】查找 206 号课程成绩不低于 101 号课程最低成绩的学生学号。

xxxxxxxxxx
SELECT 学号
FROM CJB
WHERE 课程号='206' AND 成绩 >= ANY (
    SELECT 成绩
    FROM CJB
    WHERE 课程号='101'
);

EXISTS 子查询

【例】查找选修 206 号课程的学生姓名。

xxxxxxxxxx
-- 法一：表连接（最简单）
​
SELECT 姓名
FROM XSB, CJB
WHERE XSB.学号=XJB.学号 AND 课程号='206';
​
-- 法二：子查询
​
SELECT 姓名
FROM XSB
WHERE 学号 IN (
    SELECT 学号
    FROM CJB
    WHERE 课程号='206'
);
​
-- 法三：EXISTS 子查询
​
SELECT 姓名
FROM XSB
WHERE EXISTS (
    SELECT *
    FROM CJB
    WHERE 学号=XSB.学号 AND 课程号='206'
);

【例】查找选修了全部课程的同学的姓名。

思路：不存在这样一门课程，该同学没有选修。

xxxxxxxxxx
SELECT 姓名
FROM XSB
WHERE NOT EXISTS (    -- 3. 只有该同学的未选课程的查询结果为空时，才将该同学返回
    SELECT *    -- 2. 找出该同学没有选修的全部课程
    FROM KCB
    WHERE NOT EXISTS (    -- 1. CJB 中不存在该同学对该门课程的选课记录
        SELECT *
        FROM CJB
        WHERE 学号=XSB.学号 AND 课程号=KCB.课程号
    )
);

连接

基于连接谓词的连接（内连接）

FROM 后面跟的表之间用逗号连接，连接条件写在 WHERE 后面

[INNER] JOIN: 内连接

只返回连接成功的

通常省略 INNER

【例】查找选修了 206 号课程且成绩在 80 分以上的学生姓名和成绩。

xxxxxxxxxx
SELECT 姓名, 成绩
FROM XSB JOIN CJB ON XSB.学号=CJB.学号

JOIN 后面必须有 ON（连接条件），否则错误。

（1）两个表作连接

xxxxxxxxxx
-- 写法 1: JOIN
​
SELECT 姓名, 成绩
FROM XSB JOIN CJB ON XSB.学号=CJB.学号
WHERE 课程号='206' AND 成绩 >= 80;
​
-- 写法 2: 基于连接谓词的连接
​
SELECT 姓名, 成绩
FROM XSB, CJB
WHERE XSB.学号=CJB.学号
    AND 课程号='206'
    AND 成绩 >= 80;

（2）多表连接（三个及以上的表作连接）

【例】查找选修了“计算机基础”课程且成绩在80分以上的学生的学号、姓名、课程名和成绩。

xxxxxxxxxx
-- 写法 1: JOIN
​
SELECT XSB.学号, 姓名, 课程名, 成绩
FROM XSB JOIN CJB ON XSB.学号=CJB.学号
JOIN KCB ON CJB.课程号=KCB.课程号
WHERE 课程名='计算机基础' AND 成绩 >= 80;
​
-- 写法 2: 基于连接谓词的连接
​
SELECT XSB.学号, 姓名, 课程名, 成绩
FROM XSB, CJB, KCB
WHERE 课程名='计算机基础' AND 成绩 >= 80;

另外，多表连接的 JOIN 像下面这样写也可以，不过不推荐，因为顺序容易出错：

xxxxxxxxxx
SELECT XSB.学号, 姓名, 课程名, 成绩
FROM XSB JOIN CJB JOIN KCB
    ON CJB.课程号=KCB.课程号 -- 注意顺序
    ON XSB.学号 = CJB.学号
WHERE 课程名='计算机基础' AND 成绩 >= 80;

上面这种写法中，JOIN 和 ON 必须连在一起。例如下面这种写法是错误的：

xxxxxxxxxx
SELECT XSB.学号, 姓名, 课程名, 成绩
FROM XSB JOIN CJB JOIN KCB
    ON XSB.学号 = CJB.学号 -- 顺序错误
    ON CJB.课程号=KCB.课程号
WHERE 课程名='计算机基础' AND 成绩 >= 80;

（3）自连接

【例】查找不同课程成绩相同的学生的学号、课程号和成绩。

xxxxxxxxxx
-- 写法 1: JOIN
​
SELECT a.学号, a.课程号, b.课程号, a.成绩
FROM CJB a JOIN CJB b    -- 必须取别名
ON a.成绩=b.成绩 AND a.学号=b.学号 AND a.课程号!=b.课程号;
​
-- 写法 2: 基于连接谓词的连接
​
SELECT a.学号, a.课程号, b.课程号, a.成绩
FROM CJB a, CJB b    -- 必须取别名
WHERE a.成绩=b.成绩 AND a.学号=b.学号 AND a.课程号!=b.课程号;

OUTER JOIN: 外连接

外连接：未参与连接的也列出来

  • LEFT [OUTER] JOIN
  • RIGHT [OUTER] JOIN
  • FULL [OUTER] JOIN

通常省略 OUTER。

【例】查找所有学生情况，以及他们选修的课程号，若学生未选修任何课，也要包括其情况。

xxxxxxxxxx
SELECT XSB.*, 课程号
FROM XSB LEFT JOIN CJB
ON XSB.学号=CJB.学号;

CROSS JOIN: 交叉连接

交叉连接：就是笛卡尔积。

【例】列出学生所有可能的选课情况。

xxxxxxxxxx
SELECT 学号, 姓名, 课程号, 课程名
FROM XSB CROSS JOIN KCB;

GROUP BY

若查询语句中有 GROUP BY 子句，则 SELECT 后面只能出现：

  • 出现在 GROUP BY 中的列
  • 聚合函数

【例】将 PXSCJ 数据库中各专业输出。

xxxxxxxxxx
SELECT 专业
FROM XSB
GROUP BY 专业;

上面这种写法只是想展示：GROUP BY 可以不和聚合函数一起使用。

如果只是想完成本例题要求，使用下面这种写法即可：

xxxxxxxxxx
SELECT DISTINCT 专业
FROM XSB;

【例】求各专业的学生数。

xxxxxxxxxx
SELECT 专业, COUNT(*) AS '学生数'
FROM XSB
GROUP BY 专业;

【例】求被选修的各门课程的平均成绩和选修该课程的人数，输出课程号、平均成绩和选修人数。

xxxxxxxxxx
SELECT 课程号, AVG(成绩) AS '平均成绩', COUNT(*) AS '选修人数'
FROM CJB
GROUP BY 课程号;

【例】求被选修的各门课程的平均成绩和选修该课程的人数，输出课程号、课程名、平均成绩和选修人数。

xxxxxxxxxx
SELECT CJB.课程号, 课程名, AVG(成绩) AS '平均成绩', COUNT(*) AS '选修人数'
FROM CJB JOIN KCB ON CJB.课程号=KCB.课程号
GROUP BY CJB.课程号, 课程名;    -- 注意：这里一定要有课程名！

注意：当查询中有 GROUP BY 子句时，SELECT 后面跟的列名要么在 GROUP BY 子句里，要么在聚合函数里，否则错误。

因此，上例最后一行 GROUP BY 子句里一定要有课程名。

【例】查询每个专业的男生、女生人数。

xxxxxxxxxx
SELECT 专业, 性别, COUNT(*) AS '人数'
FROM XSB
GROUP BY 专业, 性别;

【例】查询每个专业的男生、女生人数，并将结果中的 ‘0’ 和 ‘1’ 替换成 ‘女’ 和 ‘男’。

xxxxxxxxxx
SELECT 专业, 性别=
    CASE
        WHEN 性别=0 THEN '女'
        ELSE '男'
    END,
    COUNT(*) AS '人数'
FROM XSB
GROUP BY 专业, 性别;

【例】查询每一个专业的平均成绩：

xxxxxxxxxx
SELECT 专业, AVG(成绩)
FROM XSB JOIN CJB ON XSB.学号=CJB.学号
GROUP BY 专业;

HAVING

HAVING 子句与 WHERE 子句的区别是什么？

  • HAVING 子句中可以使用聚合函数，而 WHERE 子句中不可以。（也就是说，若要按照聚合函数的结果对查询结果进行进一步限制，必须放在 HAVING 后面。）
  • 使用 HAVING 则必须使用 GROUP BY。没有 GROUP BY 就不能有 HAVING。
  • HAVING 实现了对结果集的再一次筛选。

用聚合函数时不一定用 GROUP BY。使用 HAVING 时则必须有 GROUP BY。

【例】查找平均成绩在 85 分以上的学生的学号和平均成绩。

xxxxxxxxxx
SELECT 学号, AVG(成绩) AS '平均成绩'
FROM CJB
GROUP BY 学号
HAVING AVG(成绩)>=85;

【例】查找选修课程超过 2 门且成绩都在 80 分以上的学生的学号。

xxxxxxxxxx
SELECT 学号
FROM CJB
WHERE 成绩 >= 80
GROUP BY 学号
HAVING COUNT(*) > 2;

【例】查询通信工程专业平均成绩在 85 分以上的学生的学号和平均成绩。

SELECT 学号, AVG(成绩) AS '平均成绩'
FROM CJB
WHERE 学号 IN (
    SELECT 学号
    FROM XSB
    WHERE 专业='通信工程')
GROUP BY 学号
HAVING AVG(成绩) >= 85;

ORDER BY: 对查询结果排序输出

  • ASC：升序排列（默认，一般省略）
  • DESC：降序排列

【例】将计算机专业学生的“计算机基础”课程成绩按降序排列。

SELECT 姓名, 课程名, 成绩
FROM XSB, KCB, CJB
WHERE XSB.学号=CJB.学号
    AND CJB.课程号=KCB.课程号
    AND 课程名='计算机基础'
    AND 专业='计算机'
ORDER BY 成绩 DESC;

SELECT 语句的其他语法

INTO: 将查询结果保存到新表

使用 INTO 子句可以将 SELECT 查询所得的结果保存到一个新建的表中。（会真的在当前数据库中新建一个表。若同名表已存在，则报错。）

【例】由 XSB 表创建“计算机系学生”表，包括学号和姓名。

xxxxxxxxxx
SELECT 学号, 姓名
INTO 计算机系学生
FROM XSB
WHERE 专业='计算机';

UNION: 并集

使用 UNION 子句可以将两个或多个 SELECT 查询的结果合并成一个结果集。

使用 UNION 组合的两个查询的结果集需要满足如下要求：

  • 所有查询中的列数和列的顺序必须相同
  • 数据类型必须兼容

默认去重，若在 UNION 后面加 ALL 则不去重。

记住：UNION ALL 就是上下粘。

含有 UNION 的 SELECT 查询也称为联合查询。

【例】查找学号为 081101 和学号为 081210 的两位同学的信息。

SELECT * FROM XSB WHERE 学号='081101'
UNION ALL
SELECT * FROM XSB WHERE 学号='081210';

（注：本例仅为演示 UNION 子句用法，本题的“正常”解法为：）

SELECT * FROM XSB WHERE 学号 IN ('081101', '081210');

或

xxxxxxxxxx
SELECT * FROM XSB WHERE 学号='081101' OR 学号='081210';

EXCEPT: 差集

【例】查找专业为计算机但性别不为男的学生信息。

SELECT * FROM XSB WHERE 专业='计算机'
EXCEPT
SELECT * FROM XSB WHERE 性别=1;

（注：本例仅为演示 EXCEPT 子句用法，本题有更好解法）

INTERSECT：交集

【例】查找总学分大于 42 且性别为男的学生信息。

xxxxxxxxxx
SELECT * FROM XSB WHERE 总学分>42
INTERSECT
SELECT * FROM XSB WHERE 性别=1;

（注：本例仅为演示 INTERSECT 子句用法，本题有更好解法）

视图

创建视图

创建描述计算机专业学生情况的视图 CS_XS:

xxxxxxxxxx
CREATE VIEW CS_XS AS    -- 若此处不指定列名，则用 SELECT 中的
    SELECT *
    FROM XSB
    WHERE 专业='计算机'
WITH CHECK OPTION;    -- 不能通过该视图修改专业

WITH CHECK OPTION: 要求在视图上所进行的修改都要符合视图定义中 SELECT 语句所指定的限制条件。这样可以确保数据修改后，仍可通过视图看到修改后的数据。

CREATE VIEW 语句必须是批处理中的第一条语句。

【例】查找平均成绩在 80 分以上的学生的学号和平均成绩（使用视图查询而不是 HAVING 实现）。

-- 先创建学生平均成绩视图 XS_KC_AVG
CREATE VIEW XS_KC_AVG(num, score_avg) AS    -- 可以手动指定列名
    SELECT 学号, AVG(成绩)
    FROM CJB
    GROUP BY 学号;
​
-- 再对该视图进行查询，此时使用 WHERE 即可
SELECT *
FROM XS_KC_AVG
WHERE score_avg>=80;

创建视图可以向最终用户隐藏复杂的表连接，简化了用户的 SQL 程序设计。

若 SELECT 中有聚合函数，必须手动指定列名：

xxxxxxxxxx
CREATE VIEW XS_KC_AVG(num, score_avg) AS    -- 平均成绩列必须指定列名
    SELECT 学号, AVG(成绩)
    FROM CJB
    GROUP BY 学号;

分区视图——「上下粘」：

CREATE VIEW 视图名 AS
    SELECT <选择列表1> FROM T1
    UNION ALL
    SELECT <选择列表2> FROM T2
    UNION ALL
    ...
    UNION ALL
    SELECT <选择列表n> FROM Tn;

更新视图

视图一般用来查询数据，不常用来修改数据。

对视图中数据进行增、删、改的语法和普通表一样，但有如下限制：

  • 插入：对于依赖于多个基本表的视图，不能向该视图插入数据，因为这将影响多个基表。
  • 修改：对于依赖于多个基本表的视图，一次修改该视图只能变动一个基本表的数据。
  • 删除：对于依赖于多个基本表的视图，不能使用 DELETE 语句。

修改视图的定义

一般情况下，要修改视图的定义，只需删了重建即可。也可以用 ALTER VIEW 语句修改。

对于加密存储的视图不能在 SQL Server Management Studio 中通过界面修改，必须用命令行修改。

【例】视图 CS_KC 是加密存储视图，修改其定义：包括学号、姓名、选修的课程号、课程名和成绩。

xxxxxxxxxx
ALTER VIEW CS_KC WITH ENCRYPTION AS
    SELECT XSB.学号, XSB.姓名, CJB.课程号, KCB.课程名, 成绩
    FROM XSB, CJB, KCB
    WHERE CSB.学号=CJB.学号
        AND CJB.课程号=KCB.课程号
        AND 专业='计算机'
WITH CHECK OPTION;

两种修改方式的区别主要在权限上：

  • DROP VIEW -> CREATE VIEW 需要重新授权
  • ALTER VIEW 不需要重新授权

删除视图

使用 DROP VIEW 可删除一个或多个视图。

【例】删除视图 CS_XS 和 CS_KC。

xxxxxxxxxx
DROP VIEW CS_XS, CS_KC;

其他

使用视图有下列优点：

  • 为用户集中数据，简化用户的数据查询和处理
  • 屏蔽数据库的复杂性
  • 简化用户权限的管理，增加了安全性
  • 便于数据共享
  • 可以重新组织数据以便输出到其他应用程序中

T-SQL 语言

常用数据类型

整数型：推荐全用 int

精确数值型：一般用 decimal，不用浮点型，因为浮点型不精确

精确数值型包括 decimal 和 numeric 两类，这两种数据类型在功能上完全等价。

声明：numeric|decimal(p[,s])，其中 p 为精度，s 为小数位数，s 默认值为 0. 例如：decimal(6,3) 只能存储 65.343 这样的数。（小数点占一位精度）

位型：bit（只是为了省空间）

字符型：char(n), varchar(n)

Unicode 字符型：nchar(n), nvarchar(n)

二进制型：binary(n), varbinary(n)

日期时间类型：datetime（精确度为 0.03s），smalldatetime（精确到分钟）

变量

全局变量和局部变量

全局变量：以 @@ 开头，由系统提供，只能拿来用，用户不能自己定义

例如：@@ERROR 返回执行的上一个 T-SQL 语句的错误号，@@CONNECTIONS 返回自上次启动 SQL Server 以来连接或试图连接的次数，等等。

局部变量：以 @ 开头

局部变量的定义

xxxxxxxxxx
DECLARE @a CHAR(10);    -- 定义一个局部变量
DECLARE @a CHAR(10)='Alice';    -- 定义时同时赋初值
DECLARE @a INT, @b CHAR(10);    -- 定义多个局部变量

注意：即使两个变量的类型相同，声明时类型也必须写两遍，不能像 C 语言那样合并。例如下面的写法是错误的：

xxxxxxxxxx
DECLARE @a, @b INT;    -- 错误！

局部变量的赋值

可以用 SET 或 SELECT 语句为已声明的局部变量赋值：

xxxxxxxxxx
SET @a = 3;
SELECT @a = 3;

当表达式返回一个值并对一个变量进行赋值时，SET 和 SELECT 赋值没有区别，但 SQL Server 推荐使用 SET 而不是 SELECT 对变量进行赋值。

SET 与 SELECT 赋值的区别：

赋值语句前面必须有 SET 或 SELECT 其中之一，不能什么都不写。例如下面的写法是错误的：

xxxxxxxxxx
@a = 3;    -- 错误！

在使用 SET 赋值时，可以将 SELECT 语句的查询结果赋值给一个局部变量，但要求 SELECT 语句的查询结果集必须只有 1 行 1 列，否则报错：

xxxxxxxxxx
SET @a = (SELECT 性别 FROM XSB WHERE 学号='081101');

总结：使用 SELECT 语句的查询结果给局部变量赋值的两种方式（要求 SELECT 语句的查询结果集必须只有 1 行 1 列）：

xxxxxxxxxx
-- 1. 把待赋值变量放在 SELECT 查询里面
SELECT @total=SUM(学分) FROM ...
​
-- 2. 把待赋值变量放在 SELECT 查询外面
SET/SELECT @total = (SELECT SUM(学分) FROM ...)

局部变量的输出

可以用 SELECT 或 PRINT 语句输出局部变量的值：

xxxxxxxxxx
SELECT @a;
PRINT @a;

二者的区别：

  • SELECT 以表的形式输出，只有 1 行 1 列，无列名
  • PRINT 直接将变量的值打印到控制台

输出时，CHAR(n) 类型变量中存放的字符串长度若不足 n，输出时会在右边自动补全空格；VARCHAR(n) 不会自动补空格。

存储过程

简介

存储过程的分类：

  • 系统存储过程：定义在系统数据库 master 中，前缀为 sp_
  • 扩展存储过程
  • 用户存储过程（主要讲）

当需要多次重复执行某一段 SQL 代码时，把它做成存储过程效率会更高，因为存储过程只需要编译和优化一次。

存储过程中可以创建表，但不能创建视图。

创建和执行存储过程

（1）不带参数的存储过程：

【例】创建和执行存储过程：返回 081101 号学生的成绩情况。

xxxxxxxxxx
-- 创建存储过程
CREATE PROC student_info AS    -- 写全了是: CREATE PROCEDURE
    SELECT * FROM CJB WHERE 学号='081101'
    
-- 执行存储过程
EXEC student_info    -- 写全了是: EXECUTE

（2）带参数的存储过程：

【例】创建和执行存储过程：从 PXSCJ 数据库的三个表中查询某人指定课程的成绩和学分。

xxxxxxxxxx
-- 创建存储过程
CREATE PROC student_info1 @name char(8), @cname char(16) AS
    SELECT a.学号, 姓名, 课程名, 成绩, t.学分
    FROM XSB a INNER JOIN CJB b ON a.学号=b.学号
        INNER JOIN KCB t ON b.课程号=t.课程号
    WHERE a.姓名=@name AND t.课程名=@cname
    
-- 执行存储过程
EXEC student_info1 '王林','计算机基础'

（3）带 OUTPUT 参数的存储过程：

注意：定义和执行存储过程时都要写 OUTPUT。

例：创建并执行存储过程，计算指定学生所选课程的总学分。

xxxxxxxxxx
-- 创建存储过程
CREATE PROC total_credit @name varchar(40), @total int OUTPUT AS
    SELECT @total=SUM(学分) FROM XSB, CJB, KCB
    WHERE 姓名=@name AND XSB.学号=CJB.学号 AND CJB.课程号=KCB.课程号
    GROUP BY XSB.学号
    
-- 执行存储过程
DECLARE @总学分 decimal(4,1)
EXEC total_credit '王林', @总学分 OUTPUT
SELECT @总学分 AS '总学分'

（4）参数带默认值的存储过程：

xxxxxxxxxx
CREATE PROC st_info @name varchar(30)='李%' AS
    ...

存储过程和函数的区别：

  • 存储过程会提前编译好，以后执行的效率高，函数不会。
  • 函数不能有多个返回值，而存储过程可以通过 OUTPUT 参数实现多返回值。

修改存储过程

使用 ALTER PROCEDURE 修改存储过程与删除并重建存储过程的区别就在于：ALTER PROCEDURE 不用重新授权，可以保留以前赋予的许可。

语法：

xxxxxxxxxx
ALTER PROCEDURE proc_name AS
    ...(和创建存储过程的写法完全相同)

删除存储过程

xxxxxxxxxx
-- 删除前先检测一下待删除的存储过程是否已经存在，存在的话才删，避免报错
IF EXISTS(SELECT name FROM sysobjects WHERE name='proc_name')
    DROP PROCEDURE proc_name

触发器

创建触发器

按照触发事件的不同可以将触发器分为两大类：DML 触发器和 DDL 触发器。

触发器是定义在表上的。

执行触发器时，系统创建了两个特殊的临时表：inserted 表和 deleted 表，它们的表结构与 the table where the trigger is defined ON 一样。

  • 执行 INSERT 时，inserted 表中存放刚刚插入的记录。
  • 执行 DELETE 时，deleted 表中存放刚刚删除的记录。
  • UPDATE = INSERT + DELETE，执行 UPDATE 相当于先执行 DELETE 再执行 INSERT，因此 inserted 表和 deleted 表里面都会有记录。

下面的例子全都是 DML 触发器。

（1）创建 INSERT 触发器：

【例】创建触发器，当向 CJB 表中插入一个学生的成绩时，将 XSB 表中该学生的总学分加上添加的课程的学分。

xxxxxxxxxx
CREATE TRIGGER cjb_insert
ON CJB AFTER|FOR INSERT AS    -- AFTER 和 FOR 完全等价
    BEGIN
        DECLARE @num char(6), @kc_num char(3)    -- 学号, 课程号
        DECLARE @xf int    -- 学分
        SELECT @num=学号, @kc_num=课程号 from inserted    -- 取出插入的记录 *
        SELECT @xf=学分 FROM KCB WHERE 课程号=@kc_num    -- 在KCB中查询该课程的学分
        UPDATE XSB SET 总学分=总学分+@xf WHERE 学号=@num    -- 在XSB中加总学分
    END

注意：上例中标 * 的一行是假定 inserted 表中只有一条记录，也就是一次只能插入一行。若 inserted 中有多行，则只取其中一行。

（2）创建 UPDATE 触发器：

【例】创建触发器，当修改 XSB 表中的学号时，同时也要将 CJB 表中的学号修改成相应的学号（假设 XSB 表和 CJB 表之间没有定义外键约束）。

xxxxxxxxxx
CREATE TRIGGER xsb_update
ON XSB AFTER UPDATE AS
    BEGIN
        DECLARE @old_num char(6), @new_numm char(6)    -- 旧学号, 新学号
        SELECT @old_num=学号 FROM deleted
        SELECT @new_num=学号 FROM inserted
        UPDATE CJB SET 学号=@new_num WHERE 学号=@old_num
    END

（3）创建 DELETE 触发器：

【例】创建触发器，当删除 XSB 表中的一条学生记录时将 CJB 中该学生的相应记录也删除。

xxxxxxxxxx
CREATE TRIGGER xsb_delete
ON XSB AFTER DELETE AS
    BEGIN
        DECLARE @num char(6)
        SELECT @num=学号 FROM deleted
        DELETE FROM XJB WHERE 学号=@num
    END

更简洁的写法：

xxxxxxxxxx
CREATE TRIGGER xsb_delete
ON XSB AFTER DELETE AS
    BEGIN
        DELETE FROM CJB
        WHERE 学号 IN (SELECT 学号 FROM deleted)

（4）创建 DML 触发器时还可以同时创建多个类型的触发器：

例如：创建 UPDATE 和 DELETE 触发器：

xxxxxxxxxx
CREATE TRIGGER kcb_trig
ON KCB AFTER UPDATE, DELETE AS    -- 修改或删除时均触发
    ...

（5）创建 INSTEAD OF 触发器：

一个表或视图中只能有一个 INSTEAD OF 触发器。

例如，在一个表上定义了 INSTEAD OF INSERT 触发器后，在该表上执行 INSERT 语句时，并不会向表中插入数据，而是会执行该触发器中定义的内容。

写法：

xxxxxxxxxx
CREATE TRIGGER xsb_trig
ON XSB INSTEAD OF INSERT AS
    PRINT 'INSTEAD OF TRIGGER IS WORKING'

修改触发器

一般不建议修改触发器。不涉及到授权的话，直接删除重建即可。

语法：

xxxxxxxxxx
ALTER TRIGGER trigger_name
    ON ...(和创建触发器的写法完全相同)

删除触发器

xxxxxxxxxx
-- 删除前先检测一下待删除的触发器是否已经存在，存在的话才删，避免报错
IF EXISTS(SELECT name FROM sysobjects WHERE name='trigger_name')
    DROP TRIGGER trigger_name

索引

索引的分类

索引的分类：

  • 聚簇索引（聚集索引）：数据记录按聚簇索引键的次序存储。（即：数据表物理顺序与索引顺序相同。）
  • 非聚簇索引

默认：按主键建立聚簇索引。

一个表中最多只能有一个聚簇索引，但可有一个或多个非聚簇索引。

SQL Server 2005/2008 是按 B 树方式组织聚簇索引和非聚簇索引的。

当在一个表中既要创建聚簇索引，又要创建非聚簇索引时，应先创建聚簇索引，然后再创建非聚簇索引，因为创建聚簇索引时将改变数据记录的物理存放顺序。

创建索引

【例】为 KCB 表的“课程名”列创建索引。

xxxxxxxxxx
CREATE INDEX kc_name_ind ON KCB(课程名)

【例】根据 KCB 表的“课程号”列创建唯一聚簇索引。

因为指定了 CLUSTERED，所以该索引将对磁盘上的数据进行物理排序。

xxxxxxxxxx
CREATE UNIQUE CLUSTERED INDEX kc_id_ind ON KCB(课程号)

重建索引

很少用。

【例】重建 KCB 表上的 kc_name_ind 索引。

xxxxxxxxxx
ALTER INDEX kc_name_ind ON KCB REBUILD

删除索引

【例】删除 KCB 表的一个名为 kc_name_ind 的索引。

xxxxxxxxxx
-- 删除前先检测一下待删除的索引是否已经存在，存在的话才删，避免报错
IF EXISTS(SELECT name FROM sysindexes WHERE name='kc_name_ind')
    DROP INDEX KCB.kc_name_ind

数据完整性

数据完整性的分类：

  • 实体完整性（行完整性）：通过主键等实现
  • 域完整性（列完整性、用户自定义的完整性）：通过 CHECK 约束等实现
  • 参照完整性（引用完整性）：通过外键实现

【例】建立表 KCB2，同时定义总学分的约束条件为 0～60.

xxxxxxxxxx
CREATE TABLE KCB2
(
    课程号 char(6) NOT NULL,
    课程名 char(8) NOT NULL,
    学分 tinyint CHECK(学分>=0 AND 学分<=60) NULL
)
