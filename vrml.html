基本概念

VRML 源文件的扩展名是 .wrl

三个轴向：面朝观察者是 z 轴正方向，x 轴向右，y 轴向上

常用域类型：SFColor, SFFloat, SFBool, SFVec3f, SFVec2f, SFNode

Shape 节点的用法：

Shape {
    geometry Cylinder {
        # ...
    }
​
    appearance Appearance {
        material Material {
            # ...
        }
        texture ImageTexture / MovieTexture {
            # ...
        }
        textureTransform TextureTransform {
            # ...
            # 不设置时，浏览器自动拉伸填满，此时 repeatS / repeatT 无用
        }
    }
}

  • 简单造型节点：Box, Sphere, Cone, Cylinder
  • 复杂造型节点：PointSet, IndexedLineSet, IndexedFaceSet
  • 文字节点：Text

复杂造型节点

  • 不受光源影响（不能用 Appearance 中的 diffuseColor 属性设置颜色等）
  • 不能粘贴纹理
  • 不进行碰撞检测

geometry 的 color 域的优先级要高于 emissiveColor.

Transform

xxxxxxxxxx
Transform {
    children []    # 可放 Shape, Group, Transform
​
    scale 1 1 1
    rotation 0 0 1 0    # 前三个是旋转轴，最后一个是旋转弧度（逆正顺负）
    translation 0 0 0
​
    # bbox: bounding box 包围盒
    bboxCenter 0 0 0
    bboxSize -1 -1 -1    # 默认不加包围盒
​
    center 0 0 0    # 旋转轴坐标系的平移（默认是绕着当前坐标系转）
​
    # 出事件
    addChildren
    removeChildren
}

浏览器做 Transform 的顺序 (SRT)：

  1. 缩放 (scale)
  2. 旋转 (rotation)
  3. 平移 (translation)

带 children 属性的节点的基本都有如下两个属性和两个事件：

  • bboxCenter 0 0 0
  • bboxSize -1 -1 -1
  • addChildren
  • removeChildren

Material

Material {
    diffuseColor 0.8 0.8 0.8    # 漫反射颜色（最主要）

texture

纹理在所有面都会粘。

    repeatT TRUE
}

    url ""
    loop FALSE
    speed 1.0

    repeatS TRUE
    repeatT TRUE

    isActive    # 正在播放？（出事件：TRUE/FALSE）

使用 PROTO 定义节点类型和 EXTERNPROTO

调用后可更改。

xxxxxxxxxx
PROTO <NodeTypeName>
[
    # 接口域部分
    field <fieldType> <fieldName> <defaultValue>
    exposedField <fieldType> <fieldName> <defaultValue>
    eventIn <eventInType> <eventInName>
    eventOut <eventOutType> <eventOutName>
]
{
    # 节点体：定义节点执行内容
    <nodebody>
}

创建背景

VRML 中的虚拟空间背景：

  • 全景空间背景
  • 天体空间背景
xxxxxxxxxx
Background {
    # 天体空间背景
    skyColor [0.0 0.0 0.0]
    skyAngle []
    groundColor [0.0 0.0 0.0]
    groundAngle []
​
    # 全景空间背景
    frontUrl
    backUrl
    leftUrl
    rightUrl
    topUrl
    bottomUrl
​
    set_bind    # 入事件
    isBound    # 出事件
}

skyAngle:

  • 正上方0度，正下方180度
  • 第一个（0）省略，所以 skyColor 总比 skyAngle 多一个
  • 省略180度的话，最后部分全部着色为最后一个颜色
  • 必须升序排列

groundAngle:

  • 正下方0度，正上方180度
  • 如果不人为设置地面，默认只着色天空。

topUrl、bottomUrl 通常不设置背景图像，从而可以看到天空和地面。

【例】创建一个多色天空背景，由顶部的蓝色、中间的天蓝色到水平处的白色，形成一个颜色梯度变换的天空背景。

xxxxxxxxxx
Background {
    skyColor [
        0 0 1
        0 0 0.5
        1 1 1
    ]
    skyAngle [1.0 1.571]
    groundColor [
        0.1 0.1 0.1
        0.4 0.3 0.2
        0.8 0.8 0.8
    ]
    groundAngle [1.0 1.571]
}

创建光照

浏览器自带光源：白色头灯光源 headlight Bool类型

NavigationInfo 中的域，可关闭，颜色不可改变

VRML 中可人工设置的三类光源：

  • 点光源 PointLight
  • 平行光源 DirectionalLight
  • 锥光源 SpotLight（不考）

PointLight 点光源

用法：

xxxxxxxxxx
PointLight {

  • is_

入事件：监听

路由：连接入事件和出事件，出事件 -> 入事件

VRML使用六种插补器节点和八种传感器节点，通过路由在各个造型节点和场景环境节点之间传递事件

时间传感器创建关键时刻，通过路由传给插补器

关键时刻只可能出现0-1中的比例时刻

相对时刻

时间传感器 TimeSensor

和其他七个传感器都不一样，用于动画。

TimeSensor {
    enabled TRUE
    loop FALSE    # 动画是否循环
    cycleInterval 1.0    # 动画的循环周期（秒）
    startTime 0    # 一般不设置
    stopTime 0    # 一般不设置

    isActive    # 出事件 输出bool类型
    time    # 出事件 时间传感器开始运行后，不断向外输出当前时间
    cycleTime    # 出事件 时间传感器每次循环开始时输出一个当前时间
    fraction_changed # 出事件 时间传感器开始运行后，输出0到1之间的比例时刻（由插补器接收）
}

插补器

颜色插补器 ColorInterpolator

ColorInterpolator {
    key [] # 关键时刻：一系列的比例时刻
    keyValue [] # 关键状态值（颜色值）
    # key 和 keyValue 里的值一一对应。

    set_fraction # 入事件，接收时间传感器的比例时刻
    value_changed # 出事件，向外输出颜色值
}

【例】变色动画：画一个倒置的圆锥体，它将产生红、绿、蓝三色交替变换的动画。要求：循环播放，循环周期 8 秒。

#VRML V2.0 utf8

# 倒置的锥体
Transform {
    children [
        Shape {
            geometry Cone {}
            appearance Appearance {
                material DEF mm Material {
                    diffuseColor 1 0 0
                }
            }
        }
    ]
    rotation 0 0 1 3.141
}

# 变色动画

DEF time TimeSensor {
    loop TRUE
    cycleInterval 8
}

DEF cc ColorInterpolator {
    key [0 0.33 0.67 1]
    keyValue [
        1 0 0    # R
        0 1 0    # G
        0 0 1    # B
        1 0 0    # R
    ]
}

ROUTE time.fraction_changed TO cc.set_fraction
ROUTE cc.value_changed TO mm.diffuseColor

交互

用于交互的七种传感器

这七种传感器按照感知方式分为两类（每一大类中的第一个是重点）：

  • 鼠标操作：TouchSensor, PlaneSensor, CylinderSensor, SphereSensor
  • 视点位置：ProximitySensor, VisibilitySensor, Collision

TouchSensor 和 PlaneSensor 是重点

TouchSensor

TouchSensor {
    enabled    TRUE

    isOver    # 悬停 TRUE，离开 FALSE
    isActive    # 按下 TRUE，抬起 FALSE
    touchTime    # 【重要】isOver && isActive 时输出，鼠标单击就输出
    hitPoint_changed
    hitNormal_changed
    hitTexCoord_changed
}

touchTime：当 isOver=TRUE, isActive=TRUE时，引发该事件，所发送的值是当前时间。经常用此事件来控制当用户点击鼠标左键放开后触发的动作。

【例】利用 TouchSensor 触摸传感器节点，对圆锥位移动画进行控制。当光标移动到圆锥造型上时，会出现一个小手图标，表示这是一个被感应被控制的造型。此时，按下鼠标左键不放，圆锥动画开始，直到鼠标松开为止。

# VRML V2.0 utf8

DEF tt Transform {
    children [
        Shape {
            geometry Box {
                size 1 1 1
            }
            appearance Appearance {
                material Material {
                    diffuseColor 1 1 0
                }
            }
        }
    ]
}

DEF time TimeSensor {
    # 这里不能写 loop TRUE，因为 loop 现在被 TouchSensor 控制
    cycleInterval 3
}

DEF oo OrientationInterpolator {
    key [0 0.5 1]
    keyValue [
        0 1 0 0
        0 1 0 3.14    # 转半圈
        0 1 0 6.28    # 转一圈
    ]
}

ROUTE time.fraction_changed TO oo.set_fraction
ROUTE oo.value_changed TO tt.rotation

# 一般 TouchSensor 就这么写，里面什么也不写
DEF touch TouchSensor {}

# 1题
ROUTE touch.isActive TO time.loop

# 2题
ROUTE touch.touchTime TO time.startTime

注意：鼠标松开之后，动画不会立即停止，而是会播放完当前周期后才停止。
